## Answers to the audit's open questions

The audit (Section 6) asked four questions. The merge plan answers them:

1. **Adopt upstream minions end-to-end, or merge as dormant infrastructure?** End-to-end. The subsystem is additive, tested, and the CLI/MCP surface is already public in v0.12.x. Leaving it dormant would require a subtractive patch that creates drift.
2. **Do Action Brain writes depend on old `put_page` / old graph behavior?** No. Action Brain uses its own tables (`action_items`, `action_history`) and its own operations. It never calls `put_page`, never reads the links graph. Auto-link post-hook changes cannot regress it.
3. **Does our fork have Postgres data that needs v0.12.2 repair on upgrade?** Unknown but non-negotiable. Ship the `v0_12_2` orchestrator as-is; it is idempotent and self-guarding. Users who never hit the double-encode path see it no-op.
4. **Which upstream agent/skill conventions stay upstream-truth vs fork-local?** All upstream skills and conventions stay upstream-truth. The fork adds Action Brain-specific conventions (already in `src/action-brain/**`) but does not contradict or supersede upstream skills.
